# End-to-End Customer Churn Analysis

## Project Overview
Customer churn is a major challenge for telecom companies because losing customers directly impacts revenue.  
This project analyzes telecom customer data to identify churn patterns and predict customers likely to leave using SQL, Power BI, and Machine Learning.

The goal is to provide business insights that help companies improve customer retention strategies.

---

## Tools & Technologies
- SQL Server – Data cleaning and transformation
- Power BI – Interactive dashboard and KPI visualization
- Python – Machine learning model development
- Pandas & NumPy – Data processing
- Scikit-learn – Model training and evaluation

---

## Project Workflow
1. **Data Cleaning (SQL)**
   - Identified and handled missing values
   - Performed data validation and transformation

2. **Exploratory Data Analysis**
   - Analyzed churn distribution
   - Studied customer demographics and contract types

3. **Dashboard Development (Power BI)**
   - Built interactive dashboard to track churn KPIs
   - Visualized churn trends across different customer segments

4. **Machine Learning Model**
   - Random Forest classifier used for churn prediction
   - Model trained on historical customer data

5. **Prediction**
   - Model used to predict churn probability for new customers

---

## Model Performance
Algorithm Used: **Random Forest Classifier**

Model Accuracy: **~85%**

Evaluation Metrics:
- Confusion Matrix
- Classification Report

---

## Key Business Insights
- Customers with **month-to-month contracts** show the highest churn rate.
- Customers with **higher monthly charges** are more likely to churn.
- Customers with **shorter tenure** have a higher probability of leaving.
- Identified revenue impact caused by customer churn.
